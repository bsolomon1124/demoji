# CLAUDE.md

Guidance for Claude Code working in this repo. Keep it short; prefer reading
the actual files over copying them here.

## What this is

`demoji` — a small pure-Python library that finds and removes emojis in text
using Unicode Consortium emoji data bundled at build time. Ships a `demoji`
CLI and supports `python -m demoji`. Single maintainer, small user base.

## Common commands

```bash
task install       # uv sync
task test          # pytest + coverage (fails below 75%)
task lint          # ruff check
task fmt           # ruff format
task typecheck     # ty check
task build         # uv build (sdist + wheel)
task precommit     # run all pre-commit hooks via prek
task ci            # lint + fmt-check + typecheck + test + build
task refresh-codes # re-download bundled emoji data from unicode.org
```

`task --list` for the full list. Every task maps to a single `uv run` line in
`Taskfile.yml`.

## Layout

- `src/demoji/__init__.py` — public API (`findall`, `findall_list`, `replace`,
  `replace_with_desc`, `last_downloaded_timestamp`, `EMOJI_VERSION`, `CodeMap`).
- `src/demoji/main.py` — CLI entry point (`demoji = "demoji.main:main"`).
- `src/demoji/codes.json` — bundled emoji → description map (regenerated by
  `scripts/download_codes.py`).
- `src/demoji/py.typed` — PEP 561 marker; shipped in wheel.
- `tests/` — pytest suite.
- `.github/workflows/` — `ci.yml` (matrix 3.10–3.14) and `publish.yml`
  (tag-triggered: PyPI trusted publishing + Sigstore + SBOM + GH release).

## Code style

- Python 3.10+. PEP 604 unions (`X | None`), PEP 585 generics (`list[str]`).
  **Cannot** use PEP 695 `type` statement (3.12+); use `typing.TypeAlias`.
- Type-annotate public functions; `ty` runs in CI.
- Line length 88; enforced by `ruff format`.
- Emoji data is bundled — **no network access at runtime**.

## Testing

`pytest` with `pytest-cov`. Coverage threshold is **75%** (actual: 100%).
CLI-entrypoint boilerplate (`__main__.py`, `if __name__ == "__main__":`) is
marked `# pragma: no cover`.

## Repo etiquette

- Target branch: `master`.
- All actions in workflows MUST be pinned to full commit SHAs with a
  trailing `# vX.Y.Z` comment. Dependabot updates these.
- Do not commit secrets; PyPI publishing uses OIDC trusted publishing.
- Release flow: bump `version` in `pyproject.toml` and `__version__` in
  `src/demoji/__init__.py`, move `[Unreleased]` → dated heading in
  `CHANGELOG.md`, tag `vX.Y.Z`, push tag → `publish.yml` handles the rest.

## Warnings

- `set_emoji_pattern()` is a no-op kept only for backward compat — don't
  re-add caching logic.
- `_LDT` in `__init__.py` is regenerated by `scripts/download_codes.py`;
  don't hand-edit it when refreshing emoji data.
- When updating `src/demoji/codes.json`, ensure a trailing newline (the
  download script does this; manual edits must too) or the `end-of-file-fixer`
  pre-commit hook will fail.
